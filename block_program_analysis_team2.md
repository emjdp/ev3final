## 프로그램 구조 (전체 복원)

```
시작: 버튼 대기([2]=가운데) → 난수(RandomSingle 1~2 두 번 + 합) 표시 + One~Four 소리
반복 × 6구간 (구간 i = 1..6, 인터럽트 이름 01~06):
  ├─ 구간 시작(초음파 분기 True 쪽): 그리퍼 -540°(열기)
  └─ WHILE: 중앙(포트2) ColorValue 로 분기
      ├─ 검정(1): (reflect_A ≥ reflect_B − 6) ? 좌바퀴만 25% : 우바퀴만 25%   ← 지그재그
      ├─ 흰색(6): 같은 비교로 좌/우바퀴만 40%                                ← 라인 밖 복귀
      ├─ 빨강(5): Good job → 숫자(One~Six) → Red 재생(전부 완료 대기)
      │           → 제자리 유턴(탱크 90/−90, 엔코더 270°) → 타이머 표시 → 구간 탈출
      └─ 파랑(2)/무색(0): 정지 → 그리퍼 270°(집기/내리기) → Good job
                          → 그리퍼 540° → 구간 탈출   ← 통 처리 지점
구간4 뒤: 그리퍼 540° + 타이머 리셋 + 시간 LCD  ← 중간(초록) 배달로 추정
초음파(포트3): 거리 ≤ 3.8cm / 5cm 비교 → 통 감지 분기(구간별 True/False 케이스 선택)
```

## 전략 비교 (가장 중요한 통찰)

- **2조: 언롤 하드코딩.** 분기 판단이 아예 없다 — 지그재그 엣지 팔로우가
  교차로에서 자연스럽게 한쪽 팔을 타는 성질(사실상 한손법)에 기대어, 고정
  미로의 경로를 "빨강까지 가서 유턴" × 6 으로 펼쳤다. 노드 오인식이라는
  문제 자체가 없음. 색 이벤트(빨강/파랑)만 정확하면 완주.
- **우리: 일반 탐색 + 최소거리 복귀.** 미로가 바뀌어도 되고 복귀가 최단이지만,
  노드 판정·회전각·유실 복구가 전부 맞아야 한다(실패 표면이 넓다).
- 명세 4("분기점에서 자유롭게 경로 설정")는 둘 다 허용. **대회 당일 리스크를
  낮추는 관점에서, 우리 코드가 실기에서 계속 흔들리면 "2조식 언롤"을
  final_runN 의 폴백 플랜으로 고려할 것** — 우리 인프라(모션 프리미티브,
  마커 확정, 오디오)를 그대로 쓰고 Explorer 만 고정 시퀀스로 대체하면 된다.

## 그 외 관찰

- 2조 지그재그는 한쪽 바퀴를 완전히 세우는 bang-bang — 동체가 곧게 못 가고
  흔들리며 간다(사용자가 싫어한 그 주행). 그들은 저속(25%)으로 극복했다.
  우리 PID 엣지 팔로잉이 이론상 우위 — 단, 속도 과욕은 금물이라는 교훈.
- 그들 난수는 1~2 두 번 + 합산 — 스펙의 1~4 균등이 아님(2~4가 나오고 3에
  치우침). 우리 randint(1,4)가 스펙에 더 정확.
- 그들도 빨강에서 Good job 을 매번 재생 — 채점 해석 차이로 보임. 우리
  스펙 해석(good job 은 배달 시만)과 다르니 채점 규정 재확인 가치 있음.
