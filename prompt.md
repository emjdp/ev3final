# 작업: stages/final_run2.py 복귀(home) 로직 교체 — "지도 기반 최소거리 전노드 재방문"

## 배경
final_run2.py는 이미 다음 사양으로 설계/구현되어 있다(변경 금지, 그대로 유지):
- 가는 길(out): 좌>우>직 우선순위, "분기 정리형" 완전 탐색(분기 하나를 완전히
  정리하기 전엔 다음 분기로 넘어가지 않음, 새 분기 발견 시 유턴 후 직전 분기부터
  마저 정리). 이 과정에서 트리 형태의 지도(분기 노드, 각 팔의 절대 방향, 연결
  관계)가 구축된다.
- 도착(초록) 시퀀스(전진→그리퍼 오픈→후진→유턴)도 기존 그대로 유지.

## 변경 사양: 복귀(home) — 지도 기반 최소거리 "전노드 재방문"
목표가 바뀌었다: 예전(v3 방식, 경로만 되짚기)과 달리, **복귀 중에도 모든 빨강
마커를 다시 지나가야 한다**(교수님 채점 기준). 단, 총 이동거리는 최소가 되어야
한다.

### 판단층 (순수, PC 테스트 가능) — 새 함수들
1. `build_return_route(adj, mark, start, home, heading0)`:
   - `adj`: 가는 길 탐색으로 구축된 트리(노드ID → {방향: 이웃노드ID}).
   - 이 그래프는 트리이므로, "모든 노드를 방문하고 home에서 끝나는 최소거리
     경로"는 다음과 같이 유일하게 결정된다: start→home의 유일 경로(트렁크)를
     구하고, 트렁크에 있지 않은 모든 가지(막다른 길)는 트렁크상의 각 분기
     노드에서 반드시 한 번 들어갔다 나온다(왕복 2회 통과), 트렁크 자체는 한
     번만 지난다.
   - 반환값: 이동 스텝의 순서 리스트. 각 스텝은 절대 방향(N/E/S/W)과 도착
     노드ID.
   - 각 분기 노드에서 "가지(막다른 길)"가 여러 개면, 어느 걸 먼저 갈지는
     좌>우>직 우선순위(현재 heading 기준 상대 방향 계산)로 정한다. 트렁크
     진행 방향은 항상 마지막에 선택한다(그 방향을 먼저 타면 다시 못 돌아오므로).
   - 순수 재귀(DFS)로 구현: 분기 노드 도착 시 "트렁크로 계속 진행할 방향"과
     "왔던 방향"을 제외한 나머지 이웃을 좌>우>직 우선순위로 순회하며 각각
     하위 서브트리를 완전히 방문하고 복귀한 뒤, 마지막에 트렁크 방향으로
     한 걸음 전진.
   - BFS로 start→home 최단 경로(트렁크) 구하는 헬퍼 함수 별도 분리
     (`shortest_path(adj, start, home)`), 이것도 순수 함수.
2. 위 두 함수 모두 `tests/test_run_maze_v4_logic.py`에 단위 테스트 추가:
   - 이전에 검증한 코스 그래프 픽스처(빨강 6개+초록 1개)로 `build_return_route`
     실행 → (a) 반환 스텝들이 지나는 노드 집합이 빨강 6개를 모두 포함,
     (b) 트렁크 구간은 정확히 1번, 트렁크 밖 구간은 정확히 2번(왕복)씩만
     나타남, (c) 우선순위를 우>좌>직로 바꿔도 (a)(b) 동일, 방문 순서만 다름,
     (d) 총 스텝 수가 "2×(전체 간선 수) − (트렁크 간선 수)" 공식과 일치.

### 실행층 (run() 내부)
- 초록 도착 시퀀스가 끝나 유턴을 마치면(현재 v4 코드 그대로), 이 시점에
  `build_return_route`로 전체 복귀 스텝 리스트를 한 번에 계산하고 로그로
  남긴다(RETURN_PLAN, 스텝 수/예상 구간 수 포함).
- 스텝을 하나씩 실행: 절대 방향을 현재 heading과 비교해 필요한 회전
  (`_run_turn`, v2/v3 재사용) 후 `node_advance_mm` 만큼 전진 → 다음 노드
  bits/색 확인.
  - 예상한 노드(빨강/커브/트렁크 중간)와 실제 검출된 bits/색이 다르면
    RETURN_FALLBACK 로그 남기고, v3처럼 즉석 탐색(좌>우>직, 트렁크 방향
    최우선순위 하향 조정 없이 그냥 안전하게 진행)으로 전환 — 절대 그
    자리에 멈춰서 기다리지 않는다.
  - 빨강 마커 검출 시: 카운트만 증가시키고 로그(REVISIT_HOME) 남긴 뒤 계획된
    다음 스텝 계속 실행 (v3처럼 유턴하지 않는다 — 계획에 이미 유턴이
    포함되어 있음).
  - 초음파 파지 비활성(이미 내려놓음, v3 동일).
- 노랑(집) 감지 = 계획된 마지막 스텝과 일치 확인 후 정지+비프 2회 종료.
  계획보다 먼저 노랑을 만나면(폴백 경로가 지름길을 찾은 경우) 그대로 종료해도
  된다 — 단, 그 경우 몇 개 빨강을 못 봤는지 로그에 남겨라(채점 기준상 전체
  재방문이 중요하므로 조기 종료는 경고 로그 필수).

### 텔레메트리/로그
- state에 home_total(빨강 총 개수), home_revisit(재방문 카운트) 추가, _publish에 포함.
- 새 이벤트: RETURN_PLAN(전체 계획 1회, steps/segments 포함), RETURN_STEP,
  REVISIT_HOME, RETURN_FALLBACK(기존 유지), NODE_IS_HOME(완료 시 revisit
  카운트 포함하도록 필드 추가).
